# Sinkai SEOリプラン 2026-03-13

このリプランは、現状のブログ量産型SEOを「AIエージェントに現実世界実行力を足す」商談直結型SEOへ再集中させるためのものです。

## 現状認識

- ブログ公開基盤はある
- `sitemap.xml`, `robots.txt`, 記事ページの `canonical` と `Article` JSON-LD もある
- ただし、テーマが広がりすぎて `for-agents` への導線密度が落ちている
- 計測実装が見当たらず、勝ち筋の判定ができない
- ブログはグローバルナビから弱く、回遊がフッター依存になっている

## 90日方針

1. 広いAI一般論を止める
2. `現地確認`, `human-in-the-loop`, `日本語ローカル調査`, `AI出力の人間確認` に集中する
3. 記事単体ではなく、ハブページからクラスター運用に切り替える
4. 検索流入よりも `for-agents` 到達率と `quickstart` 到達率を優先指標にする

## 新規ハブページ3本

### ハブ1: 現地確認API

目的:
- 一番強い商談意図を取りに行く

掲載要素:
- できること
- 向くケース / 向かないケース
- API/MCP接続導線
- ユースケース一覧
- 関連記事

接続先:
- `/for-agents`
- `/for-agents/quickstart`
- `/for-agents/reference`

### ハブ2: AI人間レビュー設計

目的:
- human-in-the-loop 系クエリをまとめる

掲載要素:
- いつ人間レビューに切り替えるか
- 失敗時の切替フロー
- レビュー閾値
- SLAとの関係

### ハブ3: 業界別ユースケース

目的:
- 検索意図を縦に深掘る

対象候補:
- 不動産
- 日本語ローカル調査
- AI問い合わせ対応
- 現地撮影/確認

## 12週間の実行順

### Week 1-2

- ブログをグローバルナビから見える位置へ上げる
- `html lang` と記事言語整合を修正する
- Search Console / Analytics / 到達イベントを入れる
- 記事下の関連記事3件を実装する

### Week 3-4

- ハブページ3本を作る
- 01, 03, 04, 06, 46 のCTAを統一する
- 19 と 35、43b と 49 の統合方針を決める

### Week 5-8

- 更新対象の上位8本を書き換える
- `for-agents` と `quickstart` に直結する比較表を増やす
- 新規記事は月8本以下に制限する

### Week 9-12

- Search Consoleで表示回数とCTRを見てタイトル調整
- クリックは取れるがCVに弱い記事を再編集
- 優先度下げ群は内部リンク供給用として維持しつつ投資停止

## KPI

SEO単体ではなく、導線で見る:

- `blog -> for-agents` 到達率
- `blog -> quickstart` 到達率
- `blog -> ai/connect` 到達率
- ハブページのCTR
- 記事ごとの平均内部遷移数
- 商談直結キーワードの掲載順位
