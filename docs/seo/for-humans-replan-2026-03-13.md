# Sinkai SEOリプラン 2026-03-13

このリプランは、SEOの主対象を `AIエージェント導入担当` から `案件を受けるヒト` に切り替えるためのものです。

狙うユーザー:
- スマホでできる副業を探している人
- 現地確認、写真撮影、簡単調査の仕事を探している人
- スキマ時間にできる単発案件を探している人
- 在宅ではなく外出型の軽作業で稼ぎたい人

最終導線:
- `/auth`
- `/tasks`
- `/me`

## 現状認識

- 既存ブログは `for-agents` への導線が中心
- 既存記事テーマも API / MCP / AI運用 など発注者寄り
- 人向け検索意図に対して、現状のブログ内容はそのままでは噛み合わない
- したがって、今後は「記事テーマ」「CTA」「ハブページ」を人向けへ再設計する

## 90日方針

1. AI導入担当向けの一般論を止める
2. `現地確認の仕事`, `写真撮影の単発案件`, `日本語調査の副業`, `スキマ時間の仕事` に集中する
3. 記事の出口を `登録` と `案件一覧` に統一する
4. 仕事を探す人の不安解消を優先する

## コアメッセージ

`現地確認・撮影・日本語調査など、AIではなく人が必要な仕事を、単発で受けられる`

### 1. 現地確認・撮影の仕事クラスター

狙う意図:
- 現地で写真を撮る仕事を探している

### 2. 日本語調査・軽作業クラスター

狙う意図:
- 日本語調査や簡単な情報収集の案件を探している
- リモートまたは半リモートでできる仕事を探している

柱ページ候補:

- `情報収集 単発 仕事`
- `AI時代に増える人間の仕事`

### 3. 不安解消・応募前クラスター

狙う意図:
- 怪しくないか知りたい
- どうやって報酬を受け取るか知りたい

柱ページ候補:
- `現地確認の仕事とは`
- `単発案件 報酬 受け取り方`
- `写真提出 仕事 安全`
- `副業 登録だけ 仕事`

## 新規ハブページ3本

### ハブ1: 現地確認の仕事

目的:
- 最もCVに近い仕事探しクエリを取る

掲載要素:
- どんな仕事か
- 具体例
- 向いている人
- 仕事の流れ
- 登録導線
- 案件一覧導線

### ハブ2: 写真撮影・調査の単発案件

目的:
- `単発`, `副業`, `写真`, `調査` の掛け合わせを取りに行く

掲載要素:
- よくある案件
- 必要な持ち物
- 報酬イメージ
- 注意点
- 案件一覧導線

### ハブ3: はじめての人向けガイド

目的:
- 登録前の不安を潰す

掲載要素:
- 怪しくないか
- どんな作業があるか
- 支払いはどうなるか
- 未経験でもできるか
- どの地域で仕事があるか

## 12週間の実行順

### Week 1-2

- ブログのCTAを `登録` と `案件一覧` に変える
- ブログの冒頭コピーを人向けに直す
- 人向けハブの骨子を作る
- `blog -> auth`, `blog -> tasks` の導線計測を入れる

### Week 3-4

- ハブページ3本を作る
- 人向けFAQ記事を4本作る
- 既存記事から人向けに転用可能なものを選ぶ

### Week 5-8

- 仕事探し意図の強い新規記事を8本作る
- `未経験`, `安全`, `報酬`, `地域` の不安解消記事を追加する
- ブログ一覧から仕事探し導線を強化する

### Week 9-12

- CTAクリック率を見てタイトルと導線を調整する
- `blog -> auth` が弱い記事を優先的に差し替える
- エージェント向け旧記事はブランド用途として残すが主役から外す

## 実装バックログ

- `app/blog/page.tsx`: 人向け導線へ変更
- `app/blog/[slug]/page.tsx`: CTAを `登録 / 案件一覧` に変更
- `app/HomeClient.tsx`: 人向け訴求を上段へ、agent訴求を下段へ
- 新規ハブページ追加

## 既存記事の扱い

- 既存の `for-agents` 記事は即削除しない
- ただし、人向けSEOの主戦場には置かない
- 人向けの検索意図に合う記事を新規に作る
- 既存記事はブランド理解や発注者向け説明として分離運用する

## やらないこと

- API実装解説を人向けSEOの中心に置くこと
- `MCP`, `OpenAPI`, `call_human_fast` など開発者用語を人向け記事の主軸にすること
- 発注者向けと受注者向けを同じ記事で同時に説明すること
